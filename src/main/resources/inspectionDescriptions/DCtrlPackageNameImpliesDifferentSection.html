Detects packages with possibly wrong section based on the name

This inspection relies on common patterns in package names to detect possible inconsistencies between Section field and the "guessed" architecture (based on the package name). Examples include:

  • Packages ending with -doc or -docs tends to be documentation packages and should have section be "doc".
  • Packages named "aspell-<lang>" (and similar) is a common naming convention for words lists to aspell and should have section "localization".

The inspection provides a quick fix to update the section to the expected value, where it can reliably determine the correct replacement for the section value.

This inspection is based on the lintian hint wrong-section-according-to-package-name.
